h3. Forms

Forms can be set up using the @.hform@ or @.vform@ classes on any container of the input (e.g. form, fieldset, or div.text). @.col-x@ classes can be used on inputs, or if all inputs in a @fieldset@ need the same width, you can declare it on the @fieldset@.

* @.text@ -- this class goes for input[type="text"],input[type="password"], textarea 
* @.textarea@ -- include .text and .textarea
* @.select@
* @.file@
* @.checkbox@
* @.radio@
* @.checkboxes@ -- fieldset.checkboxes
* @.radios@ -- fieldset.radios
* @.hform@ -- horizontal form (labels are side by side/inline with inputs)
* @.vform@ -- vertical form (labels are on top of inputs)
* @.multi-line@ -- inputs close to gether sharing a common label (e.g. address/address line 2)
* @.fancy@ -- makes input text bold and adjusts letterspacing for a beefier look
* @.legend@ -- the legend element is a pain to style, so don't be afraid to just use a heading with the class legend.
* @.actions@ -- div.actions, a container for submit, cancel, reset buttons at the end of the form
* @.validate@ -- class to trigger JS validation (e.g. form.validate)

h3. Utility Classes

* @.reset@ -- removes most styling for elements

* @.right@ -- float right
* @.left@ -- float left

* @.center@ -- no float, auto l/r margins (needs a width, or an implied width, like an image)
