h1. Easel Guide

Easel v2 runs on "SASS 3":http://beta.sass-lang.com/docs/yardoc/file.SASS_CHANGELOG.html

To install: @gem install haml --pre@
Once installed: @sass --watch assets/styles/src:assets/styles@

h2. Naming Conventions, formatting

* CSS classes should be hyphenated, lowercase
* SASS variables and mixins should be underscored to differentiate
* 2 space soft tabs
* Classes should iterate numerically if referencing source order. Nest levels with hyphens: @.content-1@, @.content-1-1@, @.content-1-2@, @.content-2@, etc.
* Classes should iterate alphabetically if merely triggering a differnt style: @.section-c@, @.section-a@, @.section-c@,
* Avoid using ids for styling.

h2. Grid & Layout Classes

* @.section@ -- A common wrapper for most things. It is a stackable block with a base bottom margin and a clearfix applie. Furthermore, its last child should receive no margin to prevent doubling up margins.
* @.last@ -- zeros bottom margin
* @.col-x@ -- a floating column of specified width
* @.append-x@ -- append columns of padding to a column
* @.prepend-x@ -- prepend columns of padding to a column
* @.pull-x@ -- pull column to the left x columns
* @.push-x@ -- push column to the right x columns
* @.col-last@ -- last column in a row needs this to zero right margin

h2. Utility Classes

* @.js@ -- add to html element with JS to allow JS specific styling 
* @.js-hide@ -- hide if JS is enabled
* @.js-show@ -- show only if JS is enabled
* @.hide@ -- hide element with display:none
* @.hide-alt@ -- hide element, retain accessibility, does not use display (useful when JS needs to find dimensions)
* @.js-hide-alt@ -- same as above, but for JS only
* @.right@ -- float right
* @.left@ -- float left
* @.clear@ -- clear floats
* @.replace@ -- image replacement
* @.group@, @.clearfix@  -- clearfix
